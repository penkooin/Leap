Resource Management of Leap

Leap has fully-managed resource management architecture. It's divided into two different methodical part for resource service which is consisted with memory resources and file system resources.
Memory resources contributes with processing more faster response for huge amount of request and file resources is used for big size data response or non-critical on fast response as well. If a system has plenty of memory resource, It can serve for On-demand contents service and realtime streaming service like OTT or video streaming platform etc.
May if a system has a plenty of memory, In-memory resource can be having huge size of data. As a matter of fact, memory resource absolutely could provide more faster than file system.
Because of managing resources with splited part size using disassemble-assemble machanism, single huge resource can be loaded to memory.
It is able to let the server can provide single huge file to client more fast and stable. Also resource manager has proper filters to manage resources to secure and ensure.
Administrator can register filters in configuration of Leap for allowed resources and forbidden resources to be more secure and ensure to managed resource.
Resource manager provides various functionalities for supporting to combine between requested context and real resources and client can be took resources by just requesting URL on browser or others.

JPA Support of Leap

Leap has kind of methods to support using Spring/JPA component.
Most of Spring Framework's components can be used on Leap without any modification and it's functions can work well same as it's own way to use.
All of Spring Framework's annotation which for dealing with DBMS can be used on Leap service components and working well. e.g. @Service, @Entity, @Repository...
These features are achieved by put the annotations to where a profer place be needed in Leap service's type, field and method declaration.
For instance, @Autowired is annotation for injecting Spring Repository or Service object into Leap service's specifing field.
